# fs/unicode — Unicode Normalization for Filesystems

## Overview

`fs/unicode/` provides **Unicode UTF-8 normalization and case-folding** for
Linux filesystems that support case-insensitive filenames.  It is used by:

- **ext4** (`CONFIG_EXT4_UNICODE`)
- **f2fs** (`CONFIG_F2FS_FS_UNICODE`)
- **APFS** (out-of-tree)

The library implements **Unicode 12.1.0+** normalization forms using a
pre-built data table (`utf8data.c_shipped`):

- **NFC** — Canonical Decomposition followed by Canonical Composition
- **NFD** — Canonical Decomposition only
- **NFKC** — Compatibility Decomposition + Canonical Composition
- **NFKD** — Compatibility Decomposition only
- **NFDI** — NFD + case-folding (used for case-insensitive filename comparison)
- **NFKDI** — NFKD + case-folding

Source: `fs/unicode/`, `include/linux/unicode.h`.

## Subsystem Stack

```
┌────────────────────────────────────────────────────────────────┐
│                        USERSPACE                               │
│  open("Café.txt")  →  kernel receives "Caf\xc3\xa9.txt"       │
│  open("CAFE.TXT")  →  same dentry if case-insensitive enabled  │
└──────────────────────────────┬─────────────────────────────────┘
                               │  VFS lookup
┌──────────────────────────────▼─────────────────────────────────┐
│            FILESYSTEM LAYER  (ext4 / f2fs / …)                 │
│                                                                 │
│  sb->s_encoding = utf8_load(version)  ← set at mount time      │
│  sb->s_encoding_flags = ENC_FLAG_STRICT_MODE or 0              │
│                                                                 │
│  Dentry comparison: unicode_strcmp_folded(map, s1, s2)         │
│  Dentry hash:       unicode_hash_folded(map, qstr)             │
└──────────────────────────────┬─────────────────────────────────┘
                               │
┌──────────────────────────────▼─────────────────────────────────┐
│               UNICODE API  (fs/unicode/utf8-core.c)            │
│                                                                 │
│  utf8_validate(um, str)      — validate UTF-8 string           │
│  utf8_strncmp(um, s1, s2)    — normalized NFD comparison       │
│  utf8_strncasecmp(um, s1, s2)— case-folded NFDI comparison     │
│  utf8_normalize(um, str, dest) — produce normalized form       │
│  utf8_casefold(um, str, dest) — produce case-folded form       │
│                                                                 │
│  All use struct utf8cursor for streaming traversal             │
└──────────────────────────────┬─────────────────────────────────┘
                               │
┌──────────────────────────────▼─────────────────────────────────┐
│         NORMALIZATION ENGINE  (fs/unicode/utf8-norm.c)         │
│                                                                 │
│  utf8byte() — return next normalized byte from a cursor        │
│  utf8ncursor() — initialize cursor for NFC/NFD/NFKC/NFKD/NFDI │
│  Implements Unicode grapheme-cluster aware decomposition        │
│  and recomposition using lookup tables                         │
└──────────────────────────────┬─────────────────────────────────┘
                               │
┌──────────────────────────────▼─────────────────────────────────┐
│         DATA TABLES  (fs/unicode/utf8data.c_shipped)           │
│                                                                 │
│  Generated by mkutf8data.c from Unicode character database     │
│  Encodes:                                                      │
│   • Decomposition mappings (canonical + compatibility)         │
│   • Composition exclusions                                     │
│   • Case-folding tables                                        │
│   • Combining class values                                     │
│  Versioned: struct utf8data[] per Unicode version              │
└────────────────────────────────────────────────────────────────┘
```

## Normalization Forms Used by Filesystems

| Form | Used for | Description |
|---|---|---|
| NFD | Validation | Canonical decomposition; stable reference form |
| NFDI | Case-insensitive lookup | NFD + case-fold; used for dentry hash/compare |
| NFC | Display | Canonical decomposition + composition |

---

## How Case-Insensitive Lookup Works

```
open("CAFÉ.TXT")
    │
    ▼
ext4_lookup() → ext4_fname_hash(inode, &fname, hval)
    │         → unicode_hash_folded(sb->s_encoding, qstr)
    │
    ▼
utf8_casefold(um, "CAFÉ.TXT", folded_buf)
    │  → NFDI: decompose + fold → "cafe\xcc\x81.txt"
    │
    ▼
Hash table lookup → finds dentry for "café.txt"
    │
    ▼
Name comparison: utf8_strncasecmp(um, stored, incoming)
    → both normalize to same NFDI form → MATCH
```

## Key Data Structures

| Structure | Purpose |
|---|---|
| `unicode_map` | Opaque handle for a specific Unicode version + encoding |
| `utf8cursor` | Streaming cursor for normalized traversal of a UTF-8 string |
| `utf8data` | Per-Unicode-version lookup table (decomposition, case-fold) |

## Key Source Files

| File | Purpose |
|---|---|
| `fs/unicode/utf8-core.c` | Public API (validate, compare, casefold, normalize) |
| `fs/unicode/utf8-norm.c` | Normalization engine (utf8byte, utf8ncursor) |
| `fs/unicode/utf8data.c_shipped` | Pre-generated Unicode lookup tables |
| `fs/unicode/utf8n.h` | Internal declarations |
| `include/linux/unicode.h` | Public API: unicode_map, utf8_strncasecmp, etc. |
| `fs/unicode/mkutf8data.c` | Tool to regenerate utf8data.c from UCD |

## Analogy

The unicode module is like a **universal spelling normalizer for filing clerks**:

- Imagine you have a filing cabinet where every folder title must be stored in
  a single canonical spelling (NFD = decomposed form).
- When someone asks for "Café", the clerk (utf8byte/utf8cursor) first spells
  it out letter-by-letter using the canonical form (é → e + combining accent).
- For case-insensitive cabinets, the clerk also converts to lowercase before
  comparing (NFDI = NFD + case-fold), so "CAFÉ" and "café" go in the same drawer.
- The **lookup table** (`utf8data.c`) is the clerk's enormous spelling dictionary
  (Unicode character database), used to know every variant of every character.
